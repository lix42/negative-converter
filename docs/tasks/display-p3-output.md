# Display P3 Output

## Goal

Add a standards-conforming Display P3 SDR output with an embedded ICC profile,
providing the standardized destination encoding used by standalone SDR and the
planned gain-map base. Scene-to-SDR tone/reference-white rendering is owned by
`sdr-display-rendering`.

## Design

Consume already-rendered **linear Display P3** pixels from
`sdr-display-rendering`, apply Display P3's piecewise sRGB transfer function, and
attach deterministic ICC/signaling metadata. This task does not transform from
ACEScg and owns no gamut mapping or chromatic-adaptation policy in the pixel
renderer.

Synthesize the profile deterministically with Little CMS from the registered
Display P3 encoding definition:

```text
encoding white: D65 (0.3127, 0.3290)
R: (0.680, 0.320)
G: (0.265, 0.690)
B: (0.150, 0.060)
transfer: piecewise sRGB TRC (not a simple gamma 2.2)
```

Do not depend on macOS's `/System/Library/ColorSync/Profiles/Display P3.icc` or
redistribute it. Generate equivalent profile bytes cross-platform and normalize
the ICC creation timestamp as existing profiles do. Display P3 encoding is D65,
but an ICC v4 profile's PCS/media white is D50: store D50 `mediaWhitePoint`,
D65→D50 Bradford-adapted colorants plus the required `chromaticAdaptationTag`,
and verify these against the official ICC registry reference. Do not incorrectly
write D65 as the ICC media white.

Display P3 is display-referred SDR. It must not be used as nc's scanner
measurement space or linear ACEScg film master. This task owns destination color
encoding and profile metadata, not scene-to-display tone or gamut policy.

## Implementation Suggestion

- Extend the output encoding/profile enum and consume rendered linear P3.
- Add an sRGB parametric tone-curve helper rather than approximating it with a
  power curve.
- Compare profile primaries/colorants, D50 PCS/media white, chromatic adaptation,
  TRCs, and known encoded samples
  with the ICC registry reference profile; byte identity is not required.

## How to Verify

- The generated profile parses as RGB Display class with D50 PCS/media white,
  Bradford-adapted P3 colorants, the required adaptation tag, and sRGB TRCs;
  independent decoding recovers the registered D65 Display P3 encoding.
- Known rendered-linear P3 primary, neutral, and saturated samples encode within
  tolerance; tests prove this task performs no second ACEScg transform or gamut
  mapping.
- Repeated generation is byte-identical after deterministic header handling.
- TIFF output embeds the generated bytes and is recognized as Display P3 by
  independent color-managed software.
- SDR appearance is checked in macOS Preview/Photos and on an iPhone, with sRGB
  retained as the compatibility comparison.

## Dependencies

- [Color management](color-management.md)

The profile generator and linear-P3 encoding are independently testable with
synthetic destination values. Product activation waits at `sdr-display-rendering`,
which owns the ACEScg-to-rendered-linear-P3 transform and SDR gamut policy.
